## Edge Cases

### NowPlayingExpanded dismissal contract (CRITICAL regression class)
- Sheet has been dismiss-trapped 3 times historically — 5-path
  redundancy is the contract
- Each of the 5 paths must dismiss independently
- Nested sheets MUST NOT be attached (hijacks keyboard shortcuts)

### Seek echo races
- User seeks to t=120s → bridge buffers, stale POSITION_UPDATED at
  t=87s arrives 800ms later → SeekFilter drops it → lyric panel stays
  at correct line
- Cap reconcile window at 5s; after that, accept any position

### Track change navigation races
- YTM hidden WebView navigates on every track change → in-flight
  `fetch()` calls hang ~3-15 s
- Background fetches (queue refresh, lyrics preload) must wait ~1.5-2 s
  after track change so they don't saturate the channel

### Volume push settle
- After SET_VOLUME, the YTM player transiently reports volume=1.0 for
  1-2 cycles → VolumeSettle drops emits during a 2 s window
- WITHOUT this, slider snaps to MAX on track change near the volume edit

### Bridge saturation
- 150 ms poll cadence; user-driven IPCs (search, get_playlist) must not
  be starved by automatic background IPCs
- Background IPCs throttle / debounce per the rules in SWIFTUI_CHECKLIST.md

### App Group + IPC partial writes
- Extension process can be killed at any time mid-command-write
- NDJSON append records must fit in PIPE_BUF (≤512 bytes — verified for
  command shape)
- Host drainer drops unparseable lines and continues
- Stale records (>10 s) dropped during drain

### macOS rounded window corner
- Every sidebar / chrome element must clear the ~12pt rounded corner
- Verify visually — "icon visible" is not the same as "fully rendered
  without clipping"
- SWIFTUI_CHECKLIST.md has the full list of invariants

### Spotlight donation first launch
- 5000+ song libraries: donation takes 30-60 s of background CPU
- Settings status row visible during indexing
- App remains responsive

### Widget rate-limiting
- WIDGET_RELOAD_THROTTLE_MS = 2000 prevents bursty reloads from
  triggering OS rate-limiting
- Verify by playing through 5+ tracks rapidly → widget still updates
  within 2 s of each change

## Critical Paths

### Path 1: Cold launch → audio plays
1. Open VibeYTM.app from /Applications
2. App focuses correctly (no dev hack visible)
3. AuthWebView appears if not signed in; user signs into YTM
4. Once `loggedIn=true`, main UI appears with sidebar / chrome
5. User clicks a Home shelf card → song plays within 1 s
6. Now Playing widget shows cover art within 2 s
7. macOS Now Playing widget (Control Center) shows the track

### Path 2: Spotlight → play song from anywhere
1. ⌘Space → type "Despacito"
2. VibeYTM result appears in Spotlight (after Spotlight donation completes)
3. Tap → VibeYTM opens → song plays
4. Now Playing widget updates

### Path 3: Widget play/pause from outside the app
1. VibeYTM running in background
2. Click Now Playing widget Play/Pause button on Mac desktop
3. Audio toggles within ~200 ms
4. Widget icon updates to reflect new state

### Path 4: DJ Copilot full flow
1. ⌘Space → "vibe" → AppShortcut suggestion
2. Sheet opens with empty prompt
3. Type "darker vibe, 25 minutes" → submit
4. Status spinner ~3-5 s
5. Queue populates with ≥10 model-generated tracks
6. Currently-playing track switches to first suggestion
7. User can hit Like on the model's first pick (model proposed it; the
   user owns whether it stays)

### Path 5: NowPlayingExpanded open/dismiss
1. Click PlayerChrome cover thumb → expanded sheet opens
2. Try each of 5 dismissal paths — each one works
3. Lyrics overlay opens from inside expanded? NO — nested sheets are
   FORBIDDEN per SWIFTUI_CHECKLIST.md. Lyrics must dismiss expanded first.

### Path 6: Update flow (manual, per OQ1)
1. User downloads new .dmg from GitHub Releases
2. Drag to /Applications
3. First-launch Gatekeeper warning: user runs `xattr -cr /Applications/VibeYTM.app`
4. App opens; session continues (bundle ID continuity from D4)

### Path 7: Clean-machine extension registration (CRITICAL, added per Codex pushback)
1. On a Mac that has never run VibeYTM, install the .dmg (Developer-ID
   signed per D13)
2. First launch — confirm `pluginkit -m -i com.vibeytm.dev.widget`
   shows the widget extension registered with the system
3. Open Notification Center / desktop widget gallery — VibeYTM widgets
   appear and are addable
4. Add the Now Playing widget to the desktop — it renders
5. Play a song in VibeYTM — widget updates within 2 s
6. Open Shortcuts.app → VibeYTM intents appear under "Music"
7. Open Control Center → VibeYTM music tile is available
8. **Failure mode**: if any of steps 2-7 fail, the Developer-ID +
   entitlements story is broken. Common cause: App Group entitlement
   not actually provisioned in the developer profile. Surface in
   Console.app under subsystem `com.vibeytm.dev`.
